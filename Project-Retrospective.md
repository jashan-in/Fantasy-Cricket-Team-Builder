# Project Retrospective

## What went well?

Our team worked most effectively when tasks were clearly divided during sprint planning. Each member took responsibility for specific parts such as frontend features, backend development, and authentication. This helped us work in parallel and complete features efficiently. 

In the final sprint, integrating Clerk authentication and user-specific team functionality went well once roles were clearly assigned. Communication through GitHub and regular updates helped us stay aligned.

---

## What did not go well?

We faced challenges during integration between frontend and backend, especially when working with authentication and protected routes. Some issues occurred due to mismatched API routes and environment configuration.

There were also moments where work overlapped or required rework because features were not fully tested early. Deployment issues also slowed down final verification.

---

## What did we learn?

We learned how to build a full-stack application with a structured architecture and how to implement authentication using Clerk. We also gained experience in handling user-specific data securely by using session tokens and backend validation.

Additionally, we improved our skills in using GitHub for collaboration, including branching, pull requests, and resolving merge conflicts.

---

## What will we improve in future?

For future projects, we will:
- Start integration and testing earlier in the development process
- Improve communication during complex features like authentication
- Ensure better planning and clearer task breakdown during sprint planning
- Focus on debugging and deployment earlier instead of at the end
